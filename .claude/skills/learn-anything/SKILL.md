# Learn Anything — AI辅助快速学习法

> 适用于任何 AI 编程助手（Claude Code、Codex、Cursor、Gemini CLI 等）。
> 直接将本文件内容粘贴到对话中，或通过 `/learn-anything` 触发。

## 核心理念

不走线性学习路径。整个过程是**树状展开**：遇到不懂的直接问，含糊的直接展开，讨论到收敛再继续。目标：从门外汉到资深水平。

---

## 使用方式

### Claude Code
```
/learn-anything <TOPIC>
```

### 其他工具（Codex / Cursor / Gemini CLI 等）
将以下 prompt 直接发送给 AI：

```
请按照 learn-anything 工作流帮我学习 <TOPIC>。
工作流：第一步找出该领域最好的5本书；第二步根据这5本书合成一本新书（含目录和完整章节）；
第三步每章分为：通识讲解（注明书籍来源）、重点辅导（含代码和图表）、习题；
第四步我们逐章学习，遇到不懂的我会说"展开 XXX"，你立即深入讲解直到收敛，
然后更新到对应章节；习题我提交答案后你批改并更新章节。
```

---

## 工作流详解

### Phase 1：选书

用户提供 TOPIC，AI 完成：

1. 找出该领域工业界公认最好的 **5 本书**，列出：
   - 书名、作者、出版年
   - 一句话简介
   - 适读人群
   - 豆瓣 / GoodReads 评分
2. 请用户确认或替换书单后再继续

### Phase 2：合成新书

根据 5 本书内容合成一本新书：

- 先输出完整**目录**，每章标注主要来源书籍，等用户确认后再逐章生成
- 代码示例优先使用用户指定的真实项目代码（需用户提供），注明来源，写注释
- 若无真实项目代码，AI 自行生成示例并标注「AI 生成」

**章节固定结构**：

```
## 第 N 章：章节标题
> 主要来源：《书A》第X章、《书B》第Y章

### 1. 通识与知识讲解
每个知识点格式：
- 知识点名称：说明  ——《书名》第X章
- 概念之间建立关联，说明依赖关系

### 2. 重点辅导

#### 核心机制（代码示例）
```code
// 来源：项目XXX / AI生成
// 中文注释说明每一步
```

#### 辅助理解（图表）
用 ASCII 图或 Mermaid 绘制，标注「来源：《书名》」或「AI 生成」

### 3. 习题
共 N 道题，完成后达到资深水平：

1. [填空] ...
2. [简答] ...
3. [编程] ...
4. [设计] ...
```

### Phase 3：章节学习循环

每章按以下节奏迭代，直到「拿捏」：

```
[ ] 通读本章
[ ] 遇到不懂 → 说「展开 XXX」→ AI 深入讲解
[ ] 讨论收敛 → AI 更新到对应章节，附引用
[ ] 完成习题
[ ] 提交答案 + 自己的理解 → AI 批改
[ ] 指出 AI 讲解的缺失或错误 → AI 更正并完善
[ ] 批改内容追加进章节末尾（标注日期）
[ ] 本章拿捏 → 下一章
```

### Phase 4：巩固记忆

全部章节完成后，每天从头快速过一遍，直到形成长久记忆。

---

## AI 行为规则

| 触发 | AI 行为 |
|------|---------|
| 用户说「展开 XXX」 | 立即深入讲解，不限长度，直到用户说「收敛」或「继续」 |
| 展开讨论结束 | 主动问：「这个问题清楚了吗？要更新到章节吗？」 |
| 用户提交习题答案 | 给出：评分 / 逐题讲解 / 补充用户理解 / 纠正错误 |
| 习题批改完成 | 将批改内容追加到对应章节末尾，标注日期 |
| 发现书中来源 | 格式：`——《书名》第X章` |
| AI 自行生成内容 | 标注：`（AI 生成）` |

## 扩展用途

- **快速备考**：TOPIC 换成考试科目，习题改为历年真题风格
- **面试准备**：TOPIC 换成面试方向，习题改为高频面试题
- **技术选型调研**：Phase 1 改为找 5 篇最重要的论文或 RFC
